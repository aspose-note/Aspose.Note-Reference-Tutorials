# OneNote दस्तावेज़ लोड करें और संलग्नक प्राप्त करें – Aspose.Note

## परिचय

इस ट्यूटोरियल में आप सीखेंगे कि Aspose.Note for .NET के साथ **OneNote दस्तावेज़ लोड** कैसे करें और **संलग्न फ़ाइलें निकालें**। चाहे आप माइग्रेशन टूल, ऑडिट यूटिलिटी बना रहे हों, या केवल OneNote नोटबुक से संसाधन निकालने की आवश्यकता हो, नीचे दिए गए चरण आपको प्रत्येक संलग्नक को प्राप्त करने और वैकल्पिक रूप से **संलग्नक को स्ट्रीम में परिवर्तित** करने का तरीका दिखाएंगे। चलिए पूरी प्रक्रिया को देखते हैं, फ़ाइल लोड करने से लेकर प्रत्येक संलग्नक को स्थानीय रूप से सहेजने तक।

## त्वरित उत्तर
- **इस ट्यूटोरियल में क्या कवर किया गया है?** OneNote दस्तावेज़ लोड करना और उसकी संलग्न फ़ाइलें निकालना।  
- **कौन सी लाइब्रेरी आवश्यक है?** Aspose.Note for .NET (नि:शुल्क ट्रायल उपलब्ध)।  
- **कोड की कितनी पंक्तियाँ हैं?** चार संक्षिप्त स्निपेट्स में कुल 30 पंक्तियों से कम।  
- **क्या मैं संलग्नकों को स्ट्रीम कर सकता हूँ?** हाँ – उदाहरण दिखाता है कि प्रत्येक संलग्नक को मेमोरी स्ट्रीम में कैसे परिवर्तित किया जाए।  
- **क्या उत्पादन के लिए लाइसेंस चाहिए?** गैर‑ट्रायल उपयोग के लिए एक वैध Aspose.Note लाइसेंस आवश्यक है।

## “OneNote दस्तावेज़ लोड करें” क्या है?

OneNote दस्तावेज़ लोड करना का अर्थ है *.one* फ़ाइल को Aspose.Note `Document` क्लास के साथ खोलना ताकि आप प्रोग्रामेटिक रूप से उसकी सामग्री—पृष्ठ, सेक्शन, और कोई भी एम्बेडेड संसाधन जैसे संलग्न फ़ाइलें—की जाँच कर सकें।

## संलग्न फ़ाइलें क्यों निकालें?

संलग्न फ़ाइलों में अक्सर मूल्यवान संसाधन (PDF, छवियाँ, स्प्रेडशीट) होते हैं जिन्हें उपयोगकर्ता अपने नोट्स में एम्बेड करते हैं। इन्हें निकालने से आप:
- महत्वपूर्ण संसाधनों का अभिलेख या बैक‑अप बना सकते हैं।
- फ़ाइलों को आगे प्रोसेस कर सकते हैं (जैसे PDF में बदलना, सामग्री का विश्लेषण करना)।
- मैन्युअल कॉपी किए बिना अन्य अनुप्रयोगों में एसेट्स का पुनः उपयोग कर सकते हैं।

## पूर्वापेक्षाएँ

- Aspose.Note for .NET स्थापित है। आप इसे [here](https://releases.aspose.com/note/net/) से डाउनलोड कर सकते हैं।
- .NET विकास पर्यावरण (Visual Studio, VS Code, या कोई भी C# कंपाइलर)।
- एक OneNote फ़ाइल (`*.one`) जिसमें एक या अधिक संलग्न फ़ाइलें हों।

## नामस्थान आयात करें

सबसे पहले, उन नामस्थानों को आयात करें जो फ़ाइल I/O, Aspose.Note प्रकार, और कलेक्शन यूटिलिटीज़ प्रदान करते हैं:

```csharp
using System.IO;
using Aspose.Note;
using System;
using System.Collections.Generic;
using System.Drawing;
```

## चरण 1: OneNote दस्तावेज़ लोड करें

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";

// Load the document into Aspose.Note.
Document oneFile = new Document(dataDir + "Sample1.one");
```

> **प्रो टिप:** यह सुनिश्चित करें कि `dataDir` पाथ सेपरेटर (`\` या `/`) के साथ समाप्त हो ताकि गलत फ़ाइल पाथ न बनें।

## चरण 2: संलग्न फ़ाइल नोड प्राप्त करें

```csharp
// Get a list of attached file nodes
IList<AttachedFile> nodes = oneFile.GetChildNodes<AttachedFile>();
```

`GetChildNodes<AttachedFile>()` कॉल पूरे नोटबुक पदानुक्रम को स्कैन करता है और प्रत्येक `AttachedFile` तत्व को लौटाता है, जिससे आप प्रत्येक संलग्नक को व्यक्तिगत रूप से संभाल सकते हैं।

## चरण 3: संलग्न फ़ाइलों के माध्यम से पुनरावृत्ति करें और स्ट्रीम में परिवर्तित करें

```csharp
// Iterate through all nodes
foreach (AttachedFile file in nodes)
{
    // Load attached file to a stream object
    using (Stream outputStream = new MemoryStream(file.Bytes))
    {
        // Create a local file
        using (Stream fileStream = System.IO.File.OpenWrite(String.Format(dataDir + file.FileName)))
        {
            // Copy file stream
            CopyStream(outputStream, fileStream);
        }
    }
}
```

इस लूप में हम **संलग्नक को स्ट्रीम में परिवर्तित** (`MemoryStream`) करते हैं ताकि आप डेटा को मेमोरी में ही बदल सकें इससे पहले कि उसे स्थायी रूप से सहेजा जाए। `CopyStream` हेल्पर बस मेमोरी स्ट्रीम से बाइट्स को डिस्क पर एक फिजिकल फ़ाइल में कॉपी करता है।

### सामान्य समस्याएँ और समाधान
- **अनुपलब्ध अनुमतियाँ:** सुनिश्चित करें कि एप्लिकेशन को `dataDir` में लिखने की अनुमति है।
- **बड़ी संलग्नक:** बहुत बड़ी फ़ाइलों के लिए, पूरे फ़ाइल को मेमोरी में लोड करने के बजाय हिस्सों में कॉपी करने पर विचार करें।
- **फ़ाइल नाम टकराव:** यदि डुप्लिकेट नाम संभव हों तो `Path.GetUniqueFileName` का उपयोग करें या टाइमस्टैम्प जोड़ें।

## निष्कर्ष

अब आप जानते हैं कि **OneNote दस्तावेज़ लोड** करें, **संलग्न फ़ाइलें निकालें**, और **प्रत्येक संलग्नक को स्ट्रीम में परिवर्तित** करें ताकि आगे प्रोसेस किया जा सके। इन स्निपेट्स को अपने .NET प्रोजेक्ट्स में शामिल करें ताकि OneNote नोटबुक से संसाधन निकासी को स्वचालित किया जा सके।
