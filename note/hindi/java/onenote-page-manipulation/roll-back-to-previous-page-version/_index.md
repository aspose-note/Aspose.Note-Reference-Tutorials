# OneNote को कैसे रोल बैक करें - Aspose.Note

## परिचय

यदि आप किसी गलती के कारण OneNote पेज को **how to roll back onenote** करने का स्पष्ट, प्रोग्रामेटिक तरीका खोज रहे हैं, तो आप सही जगह पर हैं। इस ट्यूटोरियल में हम Aspose.Note for Java का उपयोग करके OneNote पेज को पहले की स्थिति में कैसे वापस लाएँगे, यह दिखाएंगे। चाहे आप एक ऑटोमैटिक नोट-मैनेजमेंट टूल बना रहे हों या सहयोगी नोटबुक के लिए सुरक्षा जाल की आवश्यकता हो, पेज को रोल बैक करने से आपकी सामग्री सटीक और विश्वसनीय बनी रहती है।

## त्वरित उत्तर
- **OneNote के लिए “रोल बैक” का क्या अर्थ है?** इतिहास में सहेजे गए किसी पूर्व संस्करण में पेज को पुनर्स्थापित करना।
- **कौन सा API रोलबैक को संभालता है?** Aspose.Note for Java में `PageHistory` क्लास।
- **क्या मुझे लाइसेंस की आवश्यकता है?** उत्पादन उपयोग के लिए एक वैध Aspose.Note लाइसेंस आवश्यक है।
- **क्या मैं कोई पिछला वर्शन चुन सकता हूँ?** हाँ – आप पेज के इतिहास सूची से कोई भी प्रविष्टि चुन सकते हैं।

- **क्या यह तरीका बड़ी नोटबुक के लिए सुरक्षित है?** बिल्कुल; यह ऑपरेशन अलग-अलग पेजों पर काम करता है और पूरे नोटबुक को मेमोरी में लोड नहीं करता।

## OneNote पेज वर्शन को रोल बैक कैसे करें
नीचे एक संक्षिप्त, चरण-दर-चरण गाइड दिया गया है जो दिखाता है कि रोल बैक कैसे किया जाता है। हर चरण में एक छोटा स्पष्टीकरण शामिल है ताकि आप समझ सकें *क्यों* हम यह कर रहे हैं, न कि केवल *क्या* टाइप करना है।

## ज़रूरतें

कोड में जाने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित तैयार हैं:

### Java डेवलपमेंट एनवायरनमेंट सेटअप
1. **Java डेवलपमेंट किट (JDK) इंस्टॉल करें:** Oracle वेबसाइट या अपनी पसंद के पैकेज मैनेजर से नवीनतम JDK डाउनलोड करें।
2. **Environment Variables कॉन्फ़िगर करें:** `JAVA_HOME` सेट करें और `PATH` को अपडेट करें ताकि `java` और `javac` कमांड्स कमांड लाइन से उपलब्ध हों।

3. **Java के लिए Aspose.Note जोड़ें:** लाइब्रेरी को [वेबसाइट](https://purchase.aspose.com/buy) से डाउनलोड करें और JAR को अपने प्रोजेक्ट के क्लासपाथ में जोड़ें।

## पैकेज आयात करें

OneNote फ़ाइलों के साथ काम करने के लिए आवश्यक Aspose.Note क्लासेस को इम्पोर्ट करें:

```java
import java.io.IOException;

import com.aspose.note.Document;
import com.aspose.note.Page;
import com.aspose.note.PageHistory;
```

## स्टेप 1: OneNote डॉक्यूमेंट लोड करें
```java
String dataDir = "Your Document Directory";
Document document = new Document(dataDir + "Sample1.one");
```
हम पहले उस फ़ोल्डर की ओर इशारा करते हैं जहाँ `.one` फ़ाइल स्थित है और उसे `Document` ऑब्जेक्ट में लोड करते हैं।

## स्टेप 2: पेज हिस्ट्री पाएं
```java
Page page = document.getFirstChild();
PageHistory pageHistory = document.getPageHistory(page);
```
`PageHistory` आपको चयनित पेज के प्रत्येक सहेजे गए संस्करण तक पहुँच प्रदान करता है, जिससे **restore previous onenote version** क्षमता सक्षम होती है।

## स्टेप 3: अभी का पेज हटाएं
```java
document.removeChild(page);
```
वर्तमान पेज को हटाकर हम उस संस्करण के लिए जगह बनाते हैं जिसे हम वापस लाना चाहते हैं।

## स्टेप 4: पिछला पेज वर्शन जोड़ें
```java
document.appendChildLast(pageHistory.get_Item(pageHistory.size() - 1));
```
यहाँ हम सबसे हालिया इतिहास प्रविष्टि चुनते हैं (आप इंडेक्स बदलकर किसी भी पुराने संस्करण को लक्षित कर सकते हैं) और उसे दस्तावेज़ में फिर से जोड़ते हैं।

## स्टेप 5: डॉक्यूमेंट सेव करें
```java
document.save(dataDir + "RollBackToPreviousPageVersion_out.one");
```
अंत में, संशोधित नोटबुक को सहेजें। आउटपुट फ़ाइल अब रोल‑बैक किया गया पेज रखती है।

## पिछला OneNote वर्शन रिस्टोर करें
`PageHistory` और `appendChildLast` के कॉम्बिनेशन से आप **पिछला OneNote वर्शन रिस्टोर करें** कुछ ही कोड विंडोज़ में कर सकते हैं। यह ऑटोमैटिक इंडेक्स में विशेष रूप से उपयोगी है जहाँ असाइन अनडू पॉसिबल नहीं है।

## कॉमन इश्यूज़ और टिप्स
- **एम्प्टी हिस्ट्री:** अगर `pageHistory.size()` 0 वापस करता है, तो पेज के पास कोई कॉपीरा वर्शन नहीं है—सुनिश्चित करें कि OneNote में वर्जनिंग सक्षम है।

- **इंडेक्स आउट ऑफ़ बाउंड्स:** याद रखें कि हिस्ट्री लिस्ट ज़ीरो-बेस्ड है। इच्छित वर्शन को टारगेट करने के लिए इंडेक्स (`size() - 1`) को एडजस्ट करें।

- **परफॉर्मेंस:** सिंगल पेज के साथ काम करने से पूरे नोटबुक को मेमोरी में लोड करने की ज़रूरत नहीं रहती, जिससे बड़े साइज़ के लिए भी ऑपरेशन तेज़ रहता है।

## निष्कर्ष

अब आपके पास Aspose.Note for Java का इस्तेमाल करके **how to roll back onenote** पेजों के लिए एक पूरी, प्रोडक्शन-रेडी विधि है। `PageHistory` का इस्तेमाल करके आप किसी भी नोटबुक पेज की पूर्व स्थिति को सुरक्षित रूप से पुनर्स्थापित कर सकते हैं, डेटा की अखंडता सुनिश्चित कर सकते हैं और सहयोगी वातावरण में अंतिम-उपयोग बुकमार्क को भरोसा दिला सकते हैं।

## अक्सर पूछे जाने वाले सवाल

**Q1: ​​क्या मैं पेज के कई वर्जन को रोल बैक कर सकता हूँ?**
A: हाँ, आप पूरी पेज हिस्ट्री तक पहुँच सकते हैं और आवश्यकता अनुसार किसी भी पूर्व वर्जन में रोल बैक कर सकते हैं।

**Q2: क्या Aspose.Note जावा के अलावा अन्य प्रोग्रामिंग भाषाओं को सपोर्ट करता है?**
A: हाँ, Aspose.Note विभिन्न प्रोग्रामिंग भाषाओं के लिए लाइब्रेरी देता है, जिसमें .NET, C++, और Python शामिल हैं।

**Q3: क्या Aspose.Note सभी OneNote वर्शन के साथ संगत है?**
A: Aspose.Note अलग-अलग OneNote फ़ॉर्मेट को सपोर्ट करता है, जिससे ज़्यादातर डॉक्यूमेंट वर्शन के साथ तालमेल पक्का होता है।

**Q4: क्या मैं Aspose.Note का इस्तेमाल करके OneNote में दूसरे कामों को ऑटोमैटिक कर सकता हूँ?**
A: बिल्कुल, Aspose.Note प्रोग्रामेटिक रूप से सामग्री जोड़ने, हटाने और कॉन्फ़िगर करने की विस्तृत क्षमताएँ देता है।

**Q5: क्या Aspose.Note के लिए कोई कम्युनिटी फ़ोरम या सपोर्ट उपलब्ध है?**
A: हाँ, आप [Aspose.Note फ़ोरम](https://forum.aspose.com/c/note/28) पर कम्युनिटी सपोर्ट के लिए जा सकते हैं या सहायता के लिए Aspose की कस्टमर सपोर्ट से संपर्क कर सकते हैं।
