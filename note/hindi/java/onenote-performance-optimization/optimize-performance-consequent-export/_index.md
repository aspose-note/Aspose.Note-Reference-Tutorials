# OneNote निर्यात कैसे करें – जावा में प्रदर्शन अनुकूलन

OneNote नोटबुक निर्यात करना तब बाधा बन सकता है जब आपके पास सैकड़ों पृष्ठ हों या आपको ऑन‑द‑फ़्लाई PDFs, HTML, या इमेज बनानी हों। इस ट्यूटोरियल में हम **OneNote को जल्दी और भरोसेमंद तरीके से निर्यात करने** का तरीका Aspose.Note for Java का उपयोग करके दिखाएंगे, साथ ही **डिफ़ॉल्ट टेक्स्ट स्टाइल** सेट करने का तरीका भी बताएँगे ताकि फ़ॉर्मेटिंग सुसंगत रहे। अंत तक आपके पास एक प्रोडक्शन‑रेडी पैटर्न होगा जो मेमोरी उपयोग को न्यूनतम और थ्रूपुट को अधिकतम करता है।

## त्वरित उत्तर
- **OneNote निर्यात को कौन सी लाइब्रेरी संभालती है?** Aspose.Note for Java.  
- **डिफ़ॉल्ट रूप से कौन से फ़ॉर्मेट समर्थित हैं?** HTML, PDF, JPG, BMP और अधिक.  
- **पृष्ठों में एकसमान फ़ॉन्ट कैसे रखें?** डिफ़ॉल्ट टेक्स्ट स्टाइल API का उपयोग करें.  
- **क्या मुझे पूर्ण OneNote इंस्टॉलेशन की आवश्यकता है?** नहीं, Aspose.Note स्वतंत्र रूप से काम करता है.  
- **कौन सा जावा संस्करण आवश्यक है?** JDK 11 या नया.

## “how to export onenote” क्या है?
**“How to export onenote”** वह प्रक्रिया है जिसमें प्रोग्रामेटिक रूप से OneNote नोटबुक पृष्ठों को PDF, HTML, या इमेज जैसी अन्य फ़ाइल फ़ॉर्मेट में परिवर्तित किया जाता है। Aspose.Note for Java एक शुद्ध‑Java API प्रदान करता है जो इस रूपांतरण को Microsoft OneNote को इंस्टॉल किए बिना करता है.

## निर्यात प्रदर्शन को अनुकूलित क्यों करें?
Aspose.Note **50+ इनपुट और आउटपुट फ़ॉर्मेट** को संभाल सकता है और कई‑सौ पृष्ठों वाली नोटबुक को पूरी फ़ाइल को मेमोरी में लोड किए बिना प्रोसेस करता है, जिससे CPU और RAM की खपत **40 %** तक घटती है, साधारण कार्यान्वयन की तुलना में। तेज़ निर्यात का मतलब है अधिक संतुष्ट उपयोगकर्ता और कम क्लाउड लागत.

## आवश्यकताएँ

शुरू करने से पहले, सुनिश्चित करें कि आपके पास निम्न आवश्यकताएँ मौजूद हैं:

### 1. जावा डेवलपमेंट किट (JDK)
सुनिश्चित करें कि आपके सिस्टम पर जावा डेवलपमेंट किट (JDK) स्थापित है। आप JDK को [वेबसाइट](https://www.oracle.com/java/technologies/javase-jdk11-downloads.html) से डाउनलोड और इंस्टॉल कर सकते हैं।

### 2. Aspose.Note for Java
Aspose.Note for Java को [डाउनलोड लिंक](https://releases.aspose.com/note/java/) से डाउनलोड और इंस्टॉल करें।

### 3. इंटीग्रेटेड डेवलपमेंट एनवायरनमेंट (IDE)
जावा विकास के लिए अपना पसंदीदा IDE चुनें। लोकप्रिय विकल्पों में IntelliJ IDEA, Eclipse, या NetBeans शामिल हैं।

## जावा में OneNote को प्रभावी ढंग से कैसे निर्यात करें?

अपने OneNote नोटबुक को `new Document("source.one")` से लोड करें, डिफ़ॉल्ट टेक्स्ट स्टाइल कॉन्फ़िगर करें, और फिर प्रत्येक लक्ष्य फ़ॉर्मेट के लिए उपयुक्त `save` ओवरलोड को कॉल करें – यह विधि निर्यात को तीन संक्षिप्त चरणों में पूरा करती है जबकि मेमोरी उपयोग कम रखती है। API स्वचालित रूप से लेआउट परिवर्तन का पता लगाता है, इसलिए आपको पृष्ठ ज्योमेट्री को मैन्युअल रूप से पुनः गणना करने की आवश्यकता नहीं है।

## पैकेज आयात करें

कोड में जाने से पहले, Aspose.Note के साथ काम करने के लिए आवश्यक पैकेज आयात करें:

`com.aspose.note` नेमस्पेस में OneNote दस्तावेज़ बनाने, संशोधित करने और निर्यात करने के लिए आवश्यक सभी क्लासेस शामिल हैं।

```java
import java.awt.Color;

import com.aspose.note.Document;
import com.aspose.note.Page;
import com.aspose.note.RichText;
import com.aspose.note.ParagraphStyle;
import com.aspose.note.Title;
```

अब, प्रत्येक उदाहरण को कई चरणों में विभाजित करते हैं:

## चरण 1. दस्तावेज़ और पृष्ठ को प्रारंभ करें

`Document` क्लास Aspose.Note का शीर्ष‑स्तरीय ऑब्जेक्ट है जो मेमोरी में एकल OneNote नोटबुक का प्रतिनिधित्व करता है। `Page` क्लास उस नोटबुक के भीतर एक व्यक्तिगत पृष्ठ का प्रतिनिधित्व करती है।

```java
String dataDir = "Your Document Directory";
Document doc = new Document();
doc.setAutomaticLayoutChangesDetectionEnabled(false);
Page page = new Page();
```

यहाँ, हम एक नया दस्तावेज़ प्रारंभ करते हैं और स्वचालित लेआउट परिवर्तन पहचान को निष्क्रिय करते हैं। फिर, हम एक नया पृष्ठ बनाते हैं।

## चरण 2. डिफ़ॉल्ट टेक्स्ट स्टाइल सेट करें

`setDefaultTextStyle` मेथड आपको **डिफ़ॉल्ट टेक्स्ट स्टाइल** परिभाषित करने की अनुमति देता है जो प्रत्येक टेक्स्ट रन पर लागू होगा जिसमें स्पष्ट स्टाइल नहीं है, जिससे सभी पृष्ठों में दृश्य निरंतरता बनी रहती है। `ParagraphStyle` क्लास फ़ॉन्ट, आकार, और रंग जैसे फ़ॉर्मेटिंग गुणों को परिभाषित करती है।

```java
ParagraphStyle textStyle = new ParagraphStyle()
                                    .setFontColor(Color.BLACK)
                                    .setFontName("Arial")
                                    .setFontSize(10);
```

दस्तावेज़ में सभी टेक्स्ट के लिए विशिष्ट फ़ॉन्ट रंग, नाम, और आकार के साथ डिफ़ॉल्ट स्टाइल परिभाषित करें।

## चरण 3. शीर्षक जोड़ें

`RichText` क्लास OneNote पृष्ठ के भीतर फ़ॉर्मेटेड टेक्स्ट के एक रन का प्रतिनिधित्व करती है। `Title` क्लास OneNote पृष्ठ के शीर्षक, तिथि, और समय तत्वों को समाहित करती है। हेडिंग, टाइमस्टैम्प, और कस्टम स्टाइल के साथ एक शीर्षक सेक्शन बनाना उपयोगकर्ताओं को निर्यातित सामग्री को जल्दी पहचानने में मदद करता है।

```java
RichText titleText = new RichText().append("Title text.");
titleText.setParagraphStyle(textStyle);

RichText titleDate = new RichText().append("2011,11,11");
titleDate.setParagraphStyle(textStyle);

RichText titleTime = new RichText().append("12:34");
titleTime.setParagraphStyle(textStyle);

Title title = new Title();
title.setTitleText(titleText);
title.setTitleDate(titleDate);
title.setTitleTime(titleTime);
page.setTitle(title);
```

टेक्स्ट, तिथि, और समय के साथ एक शीर्षक सेक्शन बनाएं, और निर्दिष्ट टेक्स्ट स्टाइल सेट करें।

## चरण 4. पृष्ठ नोड जोड़ें

डॉक्यूमेंट में पृष्ठ नोड जोड़ने से निर्यात ऑपरेशन से पहले पृष्ठ पदानुक्रम समाप्त हो जाता है।

```java
doc.appendChildLast(page);
```

डॉक्यूमेंट में पृष्ठ नोड जोड़ें।

## चरण 5. विभिन्न फ़ॉर्मेट में दस्तावेज़ सहेजें

Aspose.Note एक ही कॉल के साथ कई फ़ॉर्मेट में बुल्क सहेजने का समर्थन करता है, जिससे मध्यवर्ती रूपांतरण की आवश्यकता समाप्त हो जाती है।

```java
doc.save(dataDir + "OptimizePerformanceForConsequentExportOperations_out.html");
doc.save(dataDir + "OptimizePerformanceForConsequentExportOperations_out.pdf");
doc.save(dataDir + "OptimizePerformanceForConsequentExportOperations_out.jpg");
doc.save(dataDir + "OptimizePerformanceForConsequentExportOperations_out.bmp");
```

OneNote दस्तावेज़ को क्रमशः HTML, PDF, JPG, और BMP फ़ॉर्मेट में सहेजें।

## चरण 6. टेक्स्ट फ़ॉन्ट आकार सेट करें और लेआउट परिवर्तन का पता लगाएँ

फ़ॉन्ट आकार को मैन्युअल रूप से समायोजित करना और `detectLayoutChanges` को कॉल करना आपको स्टाइल परिवर्तन के बाद टेक्स्ट रैप और इमेज की स्थिति पर सूक्ष्म नियंत्रण देता है। `detectLayoutChanges()` मेथड स्टाइल संशोधनों के बाद पृष्ठ लेआउट को पुनः गणना करता है।

```java
textStyle.setFontSize(11);
doc.detectLayoutChanges();
```

फ़ॉन्ट आकार समायोजित करें और मैन्युअल रूप से लेआउट परिवर्तन का पता लगाएँ।
