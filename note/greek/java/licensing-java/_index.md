# Aspose.Note Licensing with Java – Πώς να Λάβετε τα Υπόλοιπα Credits

## Εισαγωγή

Είστε έτοιμοι να ξεκινήσετε ένα ταξίδι στον κόσμο του Aspose.Note για Java; Σε αυτόν τον ολοκληρωμένο οδηγό, θα εμβαθύνουμε στις λεπτομέρειες της διαχείρισης μετρημένων αδειών για το OneNote σε Java και θα σας δείξουμε ακριβώς πώς να **λάβετε τα υπόλοιπα credits** ώστε να διατηρείτε το κόστος υπό έλεγχο. Είτε δημιουργείτε μια πλατφόρμα SaaS, ένα εσωτερικό εργαλείο αναφοράς ή μια υπηρεσία batch‑processing, η κατανόηση της κατανάλωσης credits είναι απαραίτητη για προβλέψιμο προϋπολογισμό.

## Γρήγορες Απαντήσεις
- **Ποιος είναι ο κύριος σκοπός μιας μετρημένης άδειας;** Να σας επιτρέπει να πληρώνετε μόνο για τις κλήσεις API που χρησιμοποιείτε πραγματικά.  
- **Πώς μπορώ να παρακολουθήσω την κατανάλωση πιστώσεων;** By calling `License.setMetered` and querying the `License.getMeteredCredits()` API.  
- **Χρειάζομαι σύνδεση στο διαδίκτυο;** Ναι, η βιβλιοθήκη επικοινωνεί με τον διακομιστή αδειών της Aspose για να επικυρώσει κάθε credit.  
- **Μπορώ να μεταβώ σε μόνιμη άδεια αργότερα;** Απολύτως – απλώς αντικαταστήστε το κλειδί μετρημένης άδειας με ένα μόνιμο.  
- **Υπάρχει δωρεάν δοκιμή για τη μετρημένη άδεια;** Ναι, είναι διαθέσιμη δοκιμή 30 ημερών με περιορισμένο pool πιστώσεων.

## Τι είναι η Μετρημένη Άδεια;

Η μετρημένη άδεια είναι ένα ευέλικτο, καταναλωτικό μοντέλο που επιτρέπει στους προγραμματιστές Java να **pay‑as‑you‑go** για τις δυνατότητες του Aspose.Note. Αντί να αγοράζετε μια μόνιμη άδεια σταθερής τιμής, λαμβάνετε ένα pool credits. Κάθε φορά που καλείτε μια αδειοδοτημένη API (π.χ., δημιουργία εγγράφου OneNote, μετατροπή σελίδας), αφαιρείται ένα credit. Αυτό το μοντέλο είναι ιδανικό για λύσεις SaaS, περιστασιακές εργασίες batch ή οποιοδήποτε σενάριο όπου η χρήση κυμαίνεται.

## Γιατί να Χρησιμοποιήσετε τις Λειτουργίες Παρακολούθησης Credits του Aspose.Note;

- **Προβλεψιμότητα κόστους:** Ξοδεύετε μόνο για ό,τι χρησιμοποιείτε πραγματικά.  
- **Κλιμακωσιμότητα:** Προσθέστε περισσότερα credits κατόπιν ζήτησης χωρίς επανεγκατάσταση ή επαναδιάθεση.  
- **Ορατότητα:** Οι μετρητές credits σε πραγματικό χρόνο σας επιτρέπουν να ορίσετε ειδοποιήσεις πριν εξαντληθούν.  
- **Συμμόρφωση:** Διατηρεί την εφαρμογή σας εντός των ορίων της αγορασμένης άδειας.  
- **Λάβετε τα υπόλοιπα credits άμεσα:** Η κλήση `License.getMeteredCredits()` επιστρέφει το ακριβές υπόλοιπο, επιτρέποντας προληπτικό προϋπολογισμό.

## Προαπαιτούμενα

- Πρόσβαση στη βιβλιοθήκη Aspose.Note για Java (σύνδεσμος λήψης παρακάτω).  
- Ένα έγκυρο κλειδί μετρημένης άδειας (διαθέσιμο από το portal αγορών της Aspose).  

## Κατανόηση της Μετρημένης Άδειας

Πριν βουτήξουμε στα μαθήματα, ας κατανοήσουμε την έννοια της μετρημένης άδειας. Το Aspose.Note εισάγει τη μετρημένη άδεια για να προσφέρει μια ευέλικτη και οικονομικά αποδοτική λύση στους προγραμματιστές Java. Σας επιτρέπει να παρακολουθείτε και να ελέγχετε τη χρήση της εφαρμογής σας, διατηρώντας στενή επίβλεψη της κατανάλωσης των credits.

## Διαχείριση Μετρημένων Αδειών

### 1. Ξεκινήστε
Το πρώτο βήμα είναι να εξοικειωθείτε με τη βιβλιοθήκη Aspose.Note. Αν δεν το έχετε κάνει ήδη, [κατεβάστε](https://downloads.aspose.com/note/java) και ενσωματώστε το στο Java project σας.

### 2. Αποκτήστε μια Μετρημένη Άδεια
Αποκτήστε μια μετρημένη άδεια επισκεπτόμενοι το portal [Aspose.Purchase](https://purchase.aspose.com/). Μόλις την αποκτήσετε, εφαρμόστε την στο πρότζεκτ σας για απρόσκοπτη ενσωμάτωση.

### 3. Εφαρμόστε τη Μετρημένη Άδεια σε Java
Μάθετε πώς να εφαρμόσετε τη μετρημένη άδεια σε Java με το μάθημα για τη [διαχείριση μετρημένης άδειας για OneNote](./manage-metered-license/). Ακολουθήστε τις οδηγίες βήμα‑βήμα για να εξασφαλίσετε μια ομαλή διαδικασία ενσωμάτωσης.

## Πώς να Λάβετε τα Υπόλοιπα Credits με το Aspose.Note

Η παρακολούθηση των credits είναι τόσο απλή όσο η κλήση μερικών API μετά τον ορισμό της άδειας. Παρακάτω υπάρχει μια υψηλού επιπέδου επισκόπηση (ο πραγματικός κώδικας βρίσκεται στο συνδεδεμένο μάθημα):

1. **Ορίστε τη μετρημένη άδεια** – περάστε το κλειδί άδειας σας στο `License.setMetered`.  
2. **Εκτελέστε τις λειτουργίες OneNote** – κάθε λειτουργία αφαιρεί αυτόματα τον κατάλληλο αριθμό credits.  
3. **Ερωτήστε τα υπόλοιπα credits** – καλέστε `License.getMeteredCredits()` οποιαδήποτε στιγμή για **λάβετε τα υπόλοιπα credits** και ανακτήστε το τρέχον υπόλοιπο.  
4. **Καταγράψτε ή ειδοποιήστε** – αποθηκεύστε την τιμή στο σύστημα παρακολούθησής σας και ενεργοποιήστε ειδοποιήσεις όταν το υπόλοιπο πέσει κάτω από ένα όριο.

Ενσωματώνοντας αυτούς τους ελέγχους στη ρουτίνα παρακολούθησης υγείας της εφαρμογής σας, θα ξέρετε πάντα **πώς να λάβετε τα υπόλοιπα credits** πριν εξαντληθούν.

## Κοινά Πιθανά Σφάλματα & Συμβουλές

- **Πρόβλημα:** Ξέχνατε να καλέσετε `License.setMetered` πριν από οποιαδήποτε χρήση API.  
  **Λύση:** Αρχικοποιήστε την άδεια κατά την εκκίνηση της εφαρμογής, κατά προτίμηση σε static block.  

- **Πρόβλημα:** Δεν αντιμετωπίζετε αποτυχίες δικτύου όταν ο διακομιστής αδειών είναι μη προσβάσιμος.  
  **Λύση:** Τυλίξτε τις κλήσεις άδειας σε try‑catch blocks και επαναφέρετε τις πληροφορίες credits από cache εάν είναι δυνατόν.  

- **Pro tip:** Cache the credit count locally and only query the server once per hour to reduce latency.

## Συχνές Ερωτήσεις

**Q: Μπορώ να μεταβώ από μια μετρημένη άδεια σε μόνιμη χωρίς αλλαγές κώδικα;**  
A: Ναι. Αντικαταστήστε το κλειδί μετρημένης άδειας με ένα αρχείο μόνιμης άδειας και αφαιρέστε την κλήση `setMetered`; το υπόλοιπο του κώδικα παραμένει αμετάβλητο.

**Q: Πόσο συχνά πρέπει να ελέγχω το υπόλοιπο των credits;**  
A: Ο έλεγχος μία φορά ανά ώρα είναι συνήθως επαρκής για τις περισσότερες SaaS περιπτώσεις. Για εργασίες batch υψηλής συχνότητας, σκεφτείτε να ελέγχετε μετά από κάθε σημαντική λειτουργία.

**Q: Τι συμβαίνει αν υπερβώ το pool των credits;**  
A: Η βιβλιοθήκη ρίχνει ένα `LicenseException`. Πιάστε αυτήν την εξαίρεση για να ενημερώσετε ευγενικά τους χρήστες ή να ζητήσετε επιπλέον credits.

**Q: Υπάρχει τρόπος να αυτοματοποιήσω την ανανέωση των credits;**  
A: Η Aspose παρέχει ένα REST API για την αγορά επιπλέον credits προγραμματιστικά. Ενσωματώστε το στο admin dashboard για απρόσκοπτη κλιμάκωση.

**Q: Η παρακολούθηση credits λειτουργεί offline;**  
A: Όχι. Η επικύρωση των credits απαιτεί κλήση στο διαδίκτυο προς τον διακομιστή αδειών της Aspose. Για offline σενάρια, χρησιμοποιήστε μόνιμη άδεια.
