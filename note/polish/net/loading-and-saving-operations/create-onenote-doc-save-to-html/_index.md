# Zapisz OneNote jako HTML przy użyciu Aspose.Note

## Wprowadzenie

Jeśli potrzebujesz **save OneNote as HTML** do publikacji w sieci, raportowania lub archiwizacji, Aspose.Note dla .NET zapewnia czyste rozwiązanie wyłącznie kodowe, które działa na Windows, Linux i macOS bez wymogu posiadania Microsoft Office. W tym samouczku nauczysz się, jak od podstaw utworzyć dokument OneNote, manipulować jego stronami i wyeksportować cały notes — lub wybrane strony — do HTML przy użyciu różnych opcji. Poradnik prowadzi Cię krok po kroku, wyjaśnia, dlaczego każde ustawienie ma znaczenie, i pokazuje rzeczywiste scenariusze, w których eksport do HTML się wyróżnia, np. osadzanie treści OneNote w portalu bazy wiedzy lub generowanie statycznej dokumentacji.

## Szybkie odpowiedzi
- **Jaka jest podstawowa metoda eksportu?** Wywołaj `document.Save("output.html", SaveFormat.Html)` na instancji `Aspose.Note.Document`.  
- **Czy mogę wyeksportować pojedynczą stronę?** Tak — użyj `document.Save(pageRange, "page.html", SaveFormat.Html)`.  
- **Czy zasoby (obrazy, CSS) są osadzane automatycznie?** Domyślnie są zapisywane jako oddzielne pliki; możesz je osadzić w strumieniu pamięci przy użyciu `HtmlSaveOptions`.  
- **Czy .NET Core jest obsługiwany?** Zdecydowanie — Aspose.Note działa z .NET 5, .NET 6 i .NET Core 3.1+.  
- **Czy potrzebuję licencji do rozwoju?** Darmowa 30‑dniowa wersja próbna wystarcza do testów; licencja usuwa znak wodny oceny.

## Co to jest „save onenote as html”?

**“Save OneNote as HTML”** odnosi się do procesu konwersji notesu OneNote, przechowywanego w własnym binarnym formacie, do standardowych plików HTML, które mogą być renderowane przez dowolną przeglądarkę internetową. Konwersja zachowuje hierarchiczną strukturę notesu, formatowanie tekstu, obrazy, tabele i inne zasoby, jednocześnie tłumacząc elementy specyficzne dla OneNote na równoważne znaczniki HTML i style CSS. Umożliwia to programistom wyświetlanie treści OneNote na stronach internetowych, integrację z systemami zarządzania treścią lub archiwizację w powszechnie dostępnym formacie bez wymogu posiadania klienta OneNote.

## Dlaczego konwertować OneNote na HTML?

Aspose.Note może eksportować do **50+ formatów wyjściowych** (w tym PDF, DOCX, PNG i HTML) i przetwarza notesy z **do 500 stron** bez ładowania całego pliku do pamięci, dzięki architekturze strumieniowej. Konwersja do HTML pozwala Ci:

1. **Opublikować treść w sieci** bez wymogu posiadania klienta OneNote.  
2. **Zintegrować z systemami zarządzania treścią**, które akceptują wyłącznie HTML.  
3. **Wyodrębnić tekst możliwy do przeszukania** dla silników indeksujących.  
4. **Ponownie wykorzystać zasoby na poziomie stron** (obrazy, czcionki) w innych projektach webowych.

## Jak zapisać onenote jako html – krok po kroku

Zapis OneNote jako HTML obejmuje trzy podstawowe działania: wczytanie lub utworzenie notesu, skonfigurowanie żądanych opcji eksportu oraz wywołanie odpowiedniej metody `Save`. Najpierw tworzysz lub ładujesz obiekt `Document`, następnie możesz dostosować `HtmlSaveOptions`, aby kontrolować obsługę zasobów, generowanie CSS i kodowanie. Na końcu wywołujesz `document.Save` z podaniem ścieżki pliku lub strumienia, opcjonalnie określając `PageRange`, aby ograniczyć eksport. Takie systematyczne podejście zapewnia spójne wyniki w różnych środowiskach i pozwala dopasować wyjście do potrzeb projektu.

### Krok 1: Utwórz dokument OneNote z domyślnymi opcjami

W tym pierwszym przykładzie tworzymy nowy `Document`, dodajemy jedną stronę z tytułem i zapisujemy cały notes do HTML przy użyciu domyślnych `HtmlSaveOptions`. Nie są wymagane dodatkowe parametry, co czyni tę metodę najszybszym sposobem **save OneNote as HTML**.

```csharp
using System;
using System.Drawing;
using System.Globalization;
using System.IO;

using Aspose.Note.Saving;
using Aspose.Note.Saving.Html;

```

### Krok 2: Wyeksportuj określony zakres stron do HTML

Często potrzebny jest tylko fragment stron — np. wyeksportowanie rozdziału notatek ze spotkania. Przekazując obiekt `PageRange` do `Save`, możesz wybrać strony 1‑3, pozostawiając pozostałe niezmienione.

```csharp
public static void CreateAndSaveToHTMLUsingDefaultOptions()
{
    // Initialize OneNote document
    Document doc = new Document();
    Page page = doc.AppendChildLast(new Page());

    // Default style for all text in the document.
    ParagraphStyle textStyle = new ParagraphStyle { FontColor = Color.Black, FontName = "Arial", FontSize = 10 };
    page.Title = new Title()
    {
        TitleText = new RichText() { Text = "Title text.", ParagraphStyle = textStyle },
        TitleDate = new RichText() { Text = new DateTime(2011, 11, 11).ToString("D", CultureInfo.InvariantCulture), ParagraphStyle = textStyle },
        TitleTime = new RichText() { Text = "12:34", ParagraphStyle = textStyle }
    };

    // Save into HTML format
    string dataDir = "Your Document Directory";
    string outputPath = Path.Combine(dataDir, "CreateOneNoteDocAndSaveToHTML_out.html");
    doc.Save(outputPath);

    Console.WriteLine("\nOneNote document created successfully.\nFile saved at " + outputPath);
}
```

### Krok 3: Zapisz jako HTML do strumienia pamięci z osadzonymi zasobami

Gdy potrzebny jest pojedynczy, samodzielny ładunek HTML (np. w odpowiedziach API), osadź CSS, czcionki i obrazy bezpośrednio w `MemoryStream`. `HtmlSaveOptions` pozwala przełączyć `EmbedResources` na `true`, generując zwarty ciąg HTML.

```csharp
public static void CreateAndSavePageRange()
{
    // Initialize OneNote document
    Document doc = new Document();
    Page page = doc.AppendChildLast(new Page());

    // Default style for all text in the document.
    ParagraphStyle textStyle = new ParagraphStyle { FontColor = Color.Black, FontName = "Arial", FontSize = 10 };
    page.Title = new Title()
    {
        TitleText = new RichText() { Text = "Title text.", ParagraphStyle = textStyle },
        TitleDate = new RichText() { Text = new DateTime(2011, 11, 11).ToString("D", CultureInfo.InvariantCulture), ParagraphStyle = textStyle },
        TitleTime = new RichText() { Text = "12:34", ParagraphStyle = textStyle }
    };

    // Save into HTML format
    string dataDir = "Your Document Directory";
    string outputPath = Path.Combine(dataDir, "CreateAndSavePageRange_out.html");
    doc.Save(outputPath, new HtmlSaveOptions { PageCount = 1, PageIndex = 0 });

    Console.WriteLine("\nOneNote document created successfully and saved as page range.\nFile saved at " + outputPath);
}
```

### Krok 4: Zapisz jako HTML do plików z zasobami w osobnych plikach

Jeśli wolisz klasyczny układ folderu webowego — plik HTML plus folder zasobów — wyłącz osadzanie i określ katalog wyjściowy. To podejście daje czystsze pliki źródłowe i ułatwia buforowanie obrazów lub arkuszy stylów.

```csharp
public static void SaveAsHTMLToMemoryStreamWithEmbeddedResources()
{
    // Load the OneNote document
    string dataDir = "Your Document Directory";
    var document = new Document(Path.Combine(dataDir, "Aspose.one"));

    // Specify HTML saving options
    var options = new HtmlSaveOptions()
    {
        ExportCss = ResourceExportType.ExportEmbedded,
        ExportFonts = ResourceExportType.ExportEmbedded,
        ExportImages = ResourceExportType.ExportEmbedded,
        FontFaceTypes = FontFaceType.Ttf
    };

    // Save the document to a memory stream
    var memoryStream = new MemoryStream();
    document.Save(memoryStream, options);
}
```

### Krok 5: Zapisz jako HTML używając wywołań zwrotnych do zarządzania zasobami

Zaawansowane scenariusze, takie jak przechowywanie obrazów w bazie danych lub w chmurze, wymagają własnej obsługi. Dostarczając delegata `ResourceSavingCallback`, kontrolujesz, jak każdy zasób jest zapisywany, a API nadal generuje poprawne odwołania w HTML.

```csharp
public static void SaveAsHTMLToFileWithResourcesInSeparateFiles()
{
    // Load the OneNote document
    string dataDir = "Your Document Directory";
    var document = new Document(Path.Combine(dataDir, "Aspose.one"));

    // Specify HTML saving options
    var options = new HtmlSaveOptions()
    {
        ExportCss = ResourceExportType.ExportAsStream,
        ExportFonts = ResourceExportType.ExportAsStream,
        ExportImages = ResourceExportType.ExportAsStream,
        FontFaceTypes = FontFaceType.Ttf
    };

    // Save the document to HTML file with resources stored in separate files
    document.Save(Path.Combine(dataDir, "document_out.html"), options);
}
```

## Typowe problemy i rozwiązania

| Problem | Dlaczego się pojawia | Rozwiązanie |
|-------|----------------|-----|
| **Obrazy są uszkodzone** | Zasoby zostały zapisane w innym folderze niż plik HTML. | Upewnij się, że `HtmlSaveOptions.ResourcesFolder` odpowiada folderowi, do którego odwołujesz się, lub włącz `EmbedResources`. |
| **Brak stylów CSS** | Domyślne generowanie CSS zostało wyłączone. | Ustaw `HtmlSaveOptions.GenerateCss = true` (domyślnie) lub osadź CSS ręcznie. |
| **Duże notesy powodują OutOfMemory** | Użycie konstruktora `Document` bez strumieniowania ładuje cały plik. | Użyj `Document.Load(stream, LoadOptions)` z `LoadOptions.LoadFromMemory = true`, aby strumieniować strony. |
| **Problemy z kodowaniem HTML dla znaków nie‑ASCII** | Nieprawidłowe ustawienie `Encoding` w `HtmlSaveOptions`. | Ustaw `HtmlSaveOptions.Encoding = Encoding.UTF8`. |
| **Zapisywanie nie powodzi się w systemie Linux** | Brak natywnych zależności do przetwarzania obrazów. | Zainstaluj `libgdiplus` na hoście Linux. |

public static void SaveAsHTMLToMemoryStreamWithCallBacksToSaveResources()
{
    // Specify the saving callbacks configuration
    var savingCallbacks = new UserSavingCallbacks()
    {
        RootFolder = "documentFolder",
        CssFolder = "css",
        KeepCssStreamOpened = true,
        ImagesFolder = "images",
        FontsFolder = "fonts"
    };

    // Specify HTML saving options
    var options = new HtmlSaveOptions
    {
        FontFaceTypes = FontFaceType.Ttf,
        CssSavingCallback = savingCallbacks,
        FontSavingCallback = savingCallbacks,
        ImageSavingCallback = savingCallbacks
    };

    // Load the OneNote document
    string dataDir = "Your Document Directory";
    var document = new Document(Path.Combine(dataDir, "Aspose.one"));

    // Save the document to HTML format with resources managed by user-defined callbacks
    using (var stream = File.Create(Path.Combine(savingCallbacks.RootFolder, "document.html")))
    {
        document.Save(stream, options);
    }

    // Manually append data to the CSS stream
    using (var writer = new StreamWriter(savingCallbacks.CssStream))
    {
        writer.WriteLine();
        writer.WriteLine("/* This line is appended to stream manually by user */");
    }
}
