# วิธีการส่งออก OneNote – การเพิ่มประสิทธิภาพการทำงานใน Java

การส่งออกสมุดบันทึก OneNote อาจกลายเป็นคอขวดเมื่อคุณมีหลายร้อยหน้า หรือจำเป็นต้องสร้าง PDF, HTML หรือรูปภาพแบบเรียลไทม์ ในบทเรียนนี้เราจะสาธิต **วิธีการส่งออก OneNote** อย่างรวดเร็วและเชื่อถือได้โดยใช้ Aspose.Note for Java พร้อมกับแสดงวิธี **ตั้งค่ารูปแบบข้อความเริ่มต้น** เพื่อให้การจัดรูปแบบสอดคล้องกัน เมื่อเสร็จสิ้นคุณจะได้รูปแบบที่พร้อมใช้งานในระดับการผลิตซึ่งช่วยลดการใช้หน่วยความจำและเพิ่มอัตราการประมวลผลสูงสุด.

## คำตอบด่วน
- **ไลบรารีใดที่จัดการการส่งออก OneNote?** Aspose.Note for Java.
- **รูปแบบใดบ้างที่รองรับโดยตรง?** HTML, PDF, JPG, BMP และอื่น ๆ.
- **ฉันจะทำให้ฟอนต์คงที่ทั่วทั้งหน้าได้อย่างไร?** Use the default text style API.
- **จำเป็นต้องติดตั้ง OneNote เต็มรูปแบบหรือไม่?** No, Aspose.Note works independently.
- **ต้องการเวอร์ชัน Java ใด?** JDK 11 หรือใหม่กว่า.

## “how to export onenote” คืออะไร?
**“How to export onenote”** หมายถึงกระบวนการแปลงหน้าสมุดบันทึก OneNote เป็นรูปแบบไฟล์อื่น ๆ เช่น PDF, HTML หรือรูปภาพโดยอัตโนมัติ Aspose.Note for Java ให้ API แบบ pure‑Java ที่ทำการแปลงนี้โดยไม่ต้องติดตั้ง Microsoft OneNote

## ทำไมต้องเพิ่มประสิทธิภาพการส่งออก?
Aspose.Note สามารถจัดการ **รูปแบบเข้าและออกกว่า 50 รูปแบบ** และประมวลผลสมุดบันทึกหลายร้อยหน้าโดยไม่ต้องโหลดไฟล์ทั้งหมดเข้าสู่หน่วยความจำ ลดการใช้ CPU และ RAM ได้ถึง **40 %** เมื่อเทียบกับการทำงานแบบง่าย ๆ การส่งออกที่เร็วขึ้นหมายถึงผู้ใช้ที่พอใจและค่าใช้จ่ายคลาวด์ที่ต่ำลง.

## วิธีการส่งออก OneNote อย่างมีประสิทธิภาพใน Java?

โหลดสมุดบันทึก OneNote ของคุณด้วย `new Document("source.one")` ตั้งค่ารูปแบบข้อความเริ่มต้น แล้วเรียก `save` overload ที่เหมาะสมสำหรับแต่ละรูปแบบเป้าหมาย – วิธีนี้ทำให้การส่งออกเสร็จในสามขั้นตอนสั้น ๆ พร้อมการใช้หน่วยความจำน้อยลง API จะตรวจจับการเปลี่ยนแปลงเลย์เอาต์โดยอัตโนมัติ ดังนั้นคุณไม่จำเป็นต้องคำนวณเรขาคณิตของหน้าใหม่ด้วยตนเอง.

## นำเข้าแพ็กเกจ

ก่อนจะลงลึกในโค้ด เรามานำเข้าแพ็กเกจที่จำเป็นสำหรับการทำงานกับ Aspose.Note:

เนมสเปซ `com.aspose.note` มีคลาสทั้งหมดที่จำเป็นสำหรับการสร้าง, ปรับแต่ง, และส่งออกเอกสาร OneNote.

```java
import java.awt.Color;

import com.aspose.note.Document;
import com.aspose.note.Page;
import com.aspose.note.RichText;
import com.aspose.note.ParagraphStyle;
import com.aspose.note.Title;
```

ต่อไปนี้ เราจะแบ่งตัวอย่างแต่ละส่วนออกเป็นหลายขั้นตอน:

## ขั้นตอนที่ 1. เริ่มต้น Document และ Page

คลาส `Document` เป็นอ็อบเจ็กต์ระดับบนของ Aspose.Note ที่แสดงสมุดบันทึก OneNote หนึ่งเล่มในหน่วยความจำ คลาส `Page` แสดงหน้าต่าง ๆ ภายในสมุดบันทึกนั้น.

```java
String dataDir = "Your Document Directory";
Document doc = new Document();
doc.setAutomaticLayoutChangesDetectionEnabled(false);
Page page = new Page();
```

ที่นี่ เราเริ่มต้นเอกสารใหม่และปิดการตรวจจับการเปลี่ยนแปลงเลย์เอาต์อัตโนมัติ จากนั้นสร้างหน้าใหม่.

## ขั้นตอนที่ 2. ตั้งค่ารูปแบบข้อความเริ่มต้น

เมธอด `setDefaultTextStyle` ให้คุณกำหนด **รูปแบบข้อความเริ่มต้น** ที่จะถูกนำไปใช้กับทุกข้อความที่ไม่มีรูปแบบกำหนดไว้ล่วงหน้า เพื่อให้การแสดงผลสอดคล้องกันทั่วทั้งหน้า คลาส `ParagraphStyle` กำหนดคุณลักษณะการจัดรูปแบบเช่น ฟอนต์, ขนาด, และสีสำหรับข้อความ.

```java
ParagraphStyle textStyle = new ParagraphStyle()
                                    .setFontColor(Color.BLACK)
                                    .setFontName("Arial")
                                    .setFontSize(10);
```

กำหนดรูปแบบเริ่มต้นสำหรับข้อความทั้งหมดในเอกสารโดยระบุสีฟอนต์, ชื่อฟอนต์, และขนาดที่ต้องการ.

## ขั้นตอนที่ 3. เพิ่มหัวเรื่อง

คลาส `RichText` แสดงชุดข้อความที่จัดรูปแบบภายในหน้า OneNote คลาส `Title` รวมส่วนหัว, วันที่, และเวลาในหน้า OneNote การสร้างส่วนหัวเรื่องพร้อมหัวข้อ, เวลาประทับ, และรูปแบบที่กำหนดเองช่วยให้ผู้ใช้ระบุเนื้อหาที่ส่งออกได้อย่างรวดเร็ว.

```java
RichText titleText = new RichText().append("Title text.");
titleText.setParagraphStyle(textStyle);

RichText titleDate = new RichText().append("2011,11,11");
titleDate.setParagraphStyle(textStyle);

RichText titleTime = new RichText().append("12:34");
titleTime.setParagraphStyle(textStyle);

Title title = new Title();
title.setTitleText(titleText);
title.setTitleDate(titleDate);
title.setTitleTime(titleTime);
page.setTitle(title);
```

สร้างส่วนหัวเรื่องด้วยข้อความ, วันที่, และเวลา และตั้งค่ารูปแบบข้อความที่ระบุ.

## ขั้นตอนที่ 4. เพิ่มโหนดหน้า (Append Page Node)

การเพิ่มโหนดหน้าไปยังเอกสารทำให้โครงสร้างหน้าสมบูรณ์ก่อนทำการส่งออกใด ๆ.

```java
doc.appendChildLast(page);
```

เพิ่มโหนดหน้าไปยังเอกสาร.

## ขั้นตอนที่ 5. บันทึกเอกสารในรูปแบบต่าง ๆ

Aspose.Note รองรับการบันทึกหลายรูปแบบพร้อมกันด้วยการเรียกครั้งเดียวต่อรูปแบบ ซึ่งทำให้ไม่ต้องทำการแปลงขั้นกลาง.

```java
doc.save(dataDir + "OptimizePerformanceForConsequentExportOperations_out.html");
doc.save(dataDir + "OptimizePerformanceForConsequentExportOperations_out.pdf");
doc.save(dataDir + "OptimizePerformanceForConsequentExportOperations_out.jpg");
doc.save(dataDir + "OptimizePerformanceForConsequentExportOperations_out.bmp");
```

บันทึกเอกสาร OneNote เป็นรูปแบบ HTML, PDF, JPG, และ BMP ตามลำดับ.

## ขั้นตอนที่ 6. ตั้งค่าขนาดฟอนต์ข้อความและตรวจจับการเปลี่ยนแปลงเลย์เอาต์

การปรับขนาดฟอนต์ด้วยตนเองและเรียก `detectLayoutChanges` ให้คุณควบคุมการตัดบรรทัดของข้อความและตำแหน่งของรูปภาพหลังจากเปลี่ยนแปลงรูปแบบอย่างละเอียด เมธอด `detectLayoutChanges()` จะคำนวณเลย์เอาต์ของหน้าใหม่หลังจากการปรับรูปแบบ.

```java
textStyle.setFontSize(11);
doc.detectLayoutChanges();
```

ปรับขนาดฟอนต์และตรวจจับการเปลี่ยนแปลงเลย์เอาต์ด้วยตนเอง.

## ปัญหาทั่วไปและวิธีแก้

| ปัญหา | สาเหตุทั่วไป | วิธีแก้ |
|-------|---------------|-----|
| การส่งออกช้าเมื่อมี >200 หน้า | การตรวจจับเลย์เอาต์ทำงานบนทุกหน้า | ปิดการตรวจจับอัตโนมัติ (`setDetectLayoutChanges(false)`) และเรียก `detectLayoutChanges()` เพียงครั้งเดียวหลังจากการแก้ไขทั้งหมด |
| ฟอนต์ไม่แสดงใน PDF | ฟอนต์ไม่ได้ฝัง | ใช้ `FontSettings.setEmbedTrueTypeFonts(true)` ก่อนบันทึก คลาส `FontSettings` ควบคุมตัวเลือกการฝังฟอนต์สำหรับการส่งออก PDF |
| คุณภาพภาพต่ำ | DPI เริ่มต้นคือ 96 | ตั้งค่า `ImageSaveOptions.setResolution(300)` เพื่อให้ได้ JPG/BMP ความละเอียดสูงขึ้น คลาส `ImageSaveOptions` ระบุพารามิเตอร์การส่งออกภาพเช่น ความละเอียด |
