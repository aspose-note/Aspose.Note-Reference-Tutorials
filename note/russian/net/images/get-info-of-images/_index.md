# Извлечение метаданных изображений с помощью Aspose.Note для .NET

В этом практическом руководстве вы узнаете **как извлекать метаданные изображений** — включая ширину, высоту, оригинальный размер, имя файла и время изменения — из файлов Microsoft OneNote с использованием библиотеки Aspose.Note для C#. Независимо от того, нужно ли вам отображать миниатюры изображений, проверять размеры изображений или создавать каталог встроенных ресурсов, программное извлечение этой информации экономит бесчисленные ручные шаги.

## Быстрые ответы
- **Что означает “извлечение метаданных изображения”?** Получение свойств, таких как размеры, имя файла и метки времени, из изображений, хранящихся в документе OneNote.  
- **Какая библиотека обрабатывает это?** Aspose.Note для .NET.  
- **Нужна ли лицензия?** Бесплатная пробная версия подходит для разработки; для продакшна требуется коммерческая лицензия.  
- **Поддерживаемые платформы?** .NET Framework 4.6+, .NET Core 3.1+, .NET 5/6+.  
- **Типичное время выполнения?** Менее секунды для стандартного файла .one.

## Что такое извлечение метаданных изображения?
Извлечение метаданных изображения означает программное чтение описательных атрибутов (ширина, высота, оригинальные размеры, имя файла, время последнего изменения и т.д.), сопровождающих каждый объект изображения на странице OneNote. Эти данные полезны для проверки, отчетности или дальнейшей обработки изображений.

## Почему извлекать метаданные изображений из OneNote?
- **Автоматизация** – Создание инструментов, автоматически генерирующих инвентарь изображений.  
- **Проверка** – Убедиться, что изображения соответствуют требованиям к размеру перед публикацией.  
- **Интеграция** – Объединять контент OneNote с другими системами, которым нужны детали изображений (CMS, DAM и т.д.).  
- **Производительность** – Избегать загрузки полных бинарных данных изображений, когда нужны только их размеры.

## Требования
1. **Основы C#** – Вы должны уверенно писать базовый код на C#.  
2. **Aspose.Note для .NET** – Скачайте и установите библиотеку с официального сайта [здесь](https://releases.aspose.com/note/net/).  
3. **Файл OneNote (.one)** – Любой существующий документ OneNote, содержащий изображения.

## Импорт пространств имён
Перед началом импортируйте необходимые пространства имён:

```csharp
using System.IO;
using Aspose.Note;
using System.Collections.Generic;
using System;
```

## Шаг 1: Загрузка документа OneNote
Сначала загрузите целевой файл OneNote в объект `Aspose.Note.Document`. Это шаг **load onenote document**, подготавливающий API для дальнейших запросов.

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";

// Load the document into Aspose.Note.
Document oneFile = new Document(dataDir + "Aspose.one");
```

Замените `"Your Document Directory"` на реальную папку, в которой находится ваш файл `.one`.

## Шаг 2: Получение всех узлов изображений
Теперь мы **как извлечь изображения** получим каждый узел `Image` из документа.

```csharp
// Get all Image nodes
IList<Aspose.Note.Image> images = oneFile.GetChildNodes<Aspose.Note.Image>();
```

Метод `GetChildNodes<T>()` сканирует всю иерархию блокнота и возвращает коллекцию объектов изображений.

## Шаг 3: Итерация по каждому изображению и **c# get image properties**
Мы пройдем по коллекции и выведем метаданные, включая значения **get image dimensions** и **extract original image size**.

```csharp
foreach (Aspose.Note.Image image in images)
{
    Console.WriteLine("Width: {0}", image.Width);
    Console.WriteLine("Height: {0}", image.Height);
    Console.WriteLine("OriginalWidth: {0}", image.OriginalWidth);
    Console.WriteLine("OriginalHeight: {0}", image.OriginalHeight);
    Console.WriteLine("FileName: {0}", image.FileName);
    Console.WriteLine("LastModifiedTime: {0}", image.LastModifiedTime);
    Console.WriteLine();
}
```

Вывод показывает текущую ширину/высоту каждого изображения (как они отображаются на странице), оригинальные размеры, сохранённые в файле, имя файла и метку времени последнего изменения.

## Распространённые проблемы и решения
| Проблема | Причина | Решение |
|-------|--------|-----|
| **NullReferenceException** when `images` is empty | Документ не содержит изображений | Убедитесь, что исходный файл `.one` действительно содержит встроенные изображения. |
| **Incorrect dimensions** | Изображения масштабированы в OneNote | Используйте `OriginalWidth`/`OriginalHeight` для получения истинного размера. |
| **FileName is empty** | Изображение было вставлено из буфера обмена | API может не иметь имени файла; обработайте `null` или пустые строки в вашем коде. |

**В: Можно ли изменить свойства изображения после извлечения?**  
**О:** Да, вы можете изменить свойства, такие как `Width`, `Height` или `FileName`, а затем сохранить документ.

**В: Работает ли Aspose.Note с .NET Core?**  
**О:** Абсолютно. Библиотека полностью совместима с .NET Core, .NET 5 и .NET 6.

**В: Доступна ли бесплатная пробная версия?**  
**О:** Да, вы можете скачать пробную версию с сайта Aspose, чтобы изучить все функции.
