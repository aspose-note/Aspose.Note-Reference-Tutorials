# Загрузить документ OneNote и извлечь вложения – Aspose.Note

## Введение

В этом руководстве вы узнаете, как **загрузить документ OneNote** и **извлечь вложенные файлы** с помощью Aspose.Note для .NET. Независимо от того, создаёте ли вы инструмент миграции, утилиту аудита или просто хотите получить ресурсы из блокнота OneNote, нижеописанные шаги покажут, как получить каждое вложение и при необходимости **преобразовать вложение в поток** для дальнейшей обработки. Пройдём весь процесс от загрузки файла до сохранения каждого вложения локально.

## Быстрые ответы
- **Что покрывает это руководство?** Загрузка документа OneNote и извлечение его вложенных файлов.  
- **Какая библиотека требуется?** Aspose.Note для .NET (доступна бесплатная пробная версия).  
- **Сколько строк кода?** Менее 30 строк в четырёх лаконичных фрагментах.  
- **Можно ли потоково обрабатывать вложения?** Да — пример показывает, как преобразовать каждое вложение в поток памяти.  
- **Нужна ли лицензия для продакшна?** Для использования не в пробном режиме требуется действующая лицензия Aspose.Note.

## Что означает «загрузить документ OneNote»?
Загрузка документа OneNote подразумевает открытие файла *.one* с помощью класса Aspose.Note `Document`, чтобы программно исследовать его содержимое — страницы, разделы и любые встроенные ресурсы, такие как вложенные файлы.

## Почему стоит извлекать вложенные файлы?
Вложенные файлы часто содержат ценные ресурсы (PDF, изображения, таблицы), которые пользователи встраивают в свои заметки. Их извлечение позволяет:
- Архивировать или резервировать важные ресурсы.  
- Дальше обрабатывать файлы (например, конвертировать в PDF, анализировать содержимое).  
- Повторно использовать активы в других приложениях без ручного копирования.

## Предварительные требования

- Установлен Aspose.Note для .NET. Скачать можно [здесь](https://releases.aspose.com/note/net/).  
- Среда разработки .NET (Visual Studio, VS Code или любой компилятор C#).  
- Файл OneNote (`*.one`), содержащий одно или несколько вложенных файлов.

## Импорт пространств имён

Сначала импортируйте пространства имён, предоставляющие ввод‑вывод файлов, типы Aspose.Note и утилиты коллекций:

```csharp
using System.IO;
using Aspose.Note;
using System;
using System.Collections.Generic;
using System.Drawing;
```

## Шаг 1: Загрузить документ OneNote

```csharp
// The path to the documents directory.
string dataDir = "Your Document Directory";

// Load the document into Aspose.Note.
Document oneFile = new Document(dataDir + "Sample1.one");
```

> **Совет:** Убедитесь, что `dataDir` заканчивается разделителем пути (`\` или `/`), чтобы избежать некорректных путей к файлам.

## Шаг 2: Получить узлы вложенных файлов

```csharp
// Get a list of attached file nodes
IList<AttachedFile> nodes = oneFile.GetChildNodes<AttachedFile>();
```

Вызов `GetChildNodes<AttachedFile>()` просматривает всю иерархию блокнота и возвращает каждый элемент `AttachedFile`, позволяя обрабатывать каждое вложение отдельно.

## Шаг 3: Перебрать вложенные файлы и преобразовать в поток

```csharp
// Iterate through all nodes
foreach (AttachedFile file in nodes)
{
    // Load attached file to a stream object
    using (Stream outputStream = new MemoryStream(file.Bytes))
    {
        // Create a local file
        using (Stream fileStream = System.IO.File.OpenWrite(String.Format(dataDir + file.FileName)))
        {
            // Copy file stream
            CopyStream(outputStream, fileStream);
        }
    }
}
```

В этом цикле мы **преобразуем вложение в поток** (`MemoryStream`), чтобы можно было манипулировать данными в памяти перед их сохранением. Вспомогательный метод `CopyStream` просто копирует байты из потока памяти в физический файл на диске.

### Распространённые подводные камни и решения
- **Отсутствие прав:** Убедитесь, что приложение имеет право записи в `dataDir`.  
- **Большие вложения:** Для очень больших файлов рекомендуется копировать их частями, а не загружать полностью в память.  
- **Конфликты имён файлов:** Используйте `Path.GetUniqueFileName` или добавляйте метку времени, если возможны дублирующиеся имена.
