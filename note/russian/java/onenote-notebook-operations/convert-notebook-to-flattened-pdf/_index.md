# Как сплющить PDF из блокнота OneNote – Aspose.Note

## Введение

В этом руководстве вы узнаете, **как сплющить PDF** при конвертации блокнота OneNote в документ PDF с помощью Aspose.Note для Java. Сплющивание PDF удаляет интерактивные элементы, такие как аннотации и слои, создавая статичный файл, готовый к печати, который выглядит одинаково на любом устройстве. Независимо от того, нужно ли вам архивировать заметки встреч, делиться дизайнами с клиентами или генерировать отчёты, готовые к соблюдению нормативов, это руководство покажет точные шаги для получения чистого, сплющенного PDF за считанные минуты.

## Быстрые ответы
- **Что означает «сплющить PDF»?** Это преобразует весь интерактивный контент (аннотации, поля форм, слои) в один статичный слой страницы.  
- **Какая библиотека осуществляет конвертацию?** Aspose.Note для Java предоставляет специализированный класс `NotebookPdfSaveOptions`.  
- **Нужна ли лицензия для продакшн‑использования?** Да, требуется коммерческая лицензия; доступна бесплатная пробная версия.  
- **Можно ли пакетно обрабатывать несколько блокнотов?** Конечно – просто перебирайте файлы блокнотов и переиспользуйте те же параметры.  
- **Какая версия Java требуется?** Поддерживается Java 8 и выше.

## Что значит «как сплющить pdf» в контексте OneNote?
Сплющивание PDF означает преобразование богатого, многослойного содержимого блокнота OneNote в единый, не редактируемый поток страниц. Это особенно полезно, когда необходимо обеспечить одинаковое визуальное отображение в разных PDF‑просмотрщиках, принтерах или системах архивирования.

## Почему стоит конвертировать OneNote в PDF и сплющить его?
- **Универсальный доступ:** PDF можно открыть на любой платформе без необходимости установки OneNote.  
- **Соответствие требованиям и безопасность:** Сплющенные PDF предотвращают случайные правки и извлечение данных.  
- **Готовность к печати:** Все графики, таблицы и штрихи отображаются точно так же, как в оригинале.  
- **Удобный обмен:** Меньший размер файла и отсутствие зависимости от сервисов Microsoft.

## Предварительные требования

1. **Java Development Kit (JDK)** – установлен JDK 8 или новее.  
2. **Aspose.Note для Java** – скачайте последнюю версию [здесь](https://releases.aspose.com/note/java/).  
3. **Среда разработки (IDE)** – IntelliJ IDEA, Eclipse или любой другой предпочитаемый редактор.  

## Импорт пакетов

Сначала импортируйте необходимые классы, которые позволят работать с блокнотами и параметрами сохранения PDF:

```java
import com.aspose.note.Notebook;
import com.aspose.note.NotebookPdfSaveOptions;
import java.io.IOException;
```

## Шаг 1: Загрузка блокнота OneNote

Загрузите блокнот, который хотите конвертировать. Замените путь‑заполнитель реальным расположением вашего файла `.onetoc2`.

```java
String dataDir = "Your Document Directory";
Notebook notebook = new Notebook(dataDir + "Notizbuch öffnen.onetoc2");
```

## Шаг 2: Установка параметров конвертации (сплющивание PDF)

Создайте экземпляр `NotebookPdfSaveOptions` и включите флаг `flatten`. Это указывает Aspose.Note генерировать сплющенный PDF.

```java
NotebookPdfSaveOptions notebookSaveOptions = new NotebookPdfSaveOptions();
notebookSaveOptions.setFlatten(true);
```

## Шаг 3: Сохранение блокнота как сплющенного PDF

Определите имя выходного файла и вызовите метод `save`, передав сконфигурированные параметры.

```java
dataDir = dataDir + "ExportNotebookToPDFAsFlattened_out.pdf";
// Save the Notebook
notebook.save(dataDir, notebookSaveOptions);
```

### Как конвертировать блокнот в PDF (без сплющивания) – опционально
Если вам нужен обычный (не сплющенный) PDF, просто установите `setFlatten(false)` или полностью опустите вызов. Один и тот же API поддерживает оба сценария.

## Распространённые проблемы и решения

| **Исключение «File not found»** | Неправильный путь `dataDir` | Проверьте абсолютный путь или используйте `Paths.get(...)` для кроссплатформенных путей. |

## Часто задаваемые вопросы

**В: Как сплющить PDF, сохранив качество изображений?**  
О: Класс `NotebookPdfSaveOptions` сохраняет оригинальное разрешение изображений; просто не уменьшайте масштаб изображений перед сохранением.

**В: Можно ли добавить пароль к сплющенному PDF?**  
О: Да, установите свойство `setPassword` у `NotebookPdfSaveOptions` перед вызовом `save`.

**В: Можно ли внедрить пользовательские метаданные в PDF?**  
О: Используйте метод `setMetadata` у `NotebookPdfSaveOptions` для добавления заголовка, автора и других полей метаданных PDF.

**В: Будут ли аннотации видимы после сплющивания?**  
О: Аннотации становятся частью графики страницы, поэтому они остаются видимыми, но уже не редактируемыми.

**В: Влияет ли сплющивание на размер файла?**  
О: Обычно сплющивание уменьшает размер файла, так как интерактивные объекты удаляются, но крупные встроенные изображения могут сохранять большой размер.
