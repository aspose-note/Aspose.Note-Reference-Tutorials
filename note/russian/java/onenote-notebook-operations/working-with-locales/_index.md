# Установить локаль по умолчанию в OneNote – Aspose.Note Java

## Введение

Если вам необходимо **установить локаль по умолчанию** при обработке файлов OneNote, Aspose.Note для Java делает это проще простого. В этом руководстве мы пройдёмся по всем шагам — от лицензирования продукта до загрузки документа OneNote, изменения локали и, наконец, конвертации файла в изображение PNG. К концу вы сможете настроить языковые параметры и генерировать вывод, зависящий от локали, всего несколькими строками кода.

## Краткие ответы
- **Что делает «установить локаль по умолчанию»?** Определяет язык и региональные форматы, используемые Aspose.Note при чтении или записи файлов OneNote.  
- **Нужна ли лицензия?** Да — установите лицензию Aspose, чтобы разблокировать полную функциональность.  
- **Какая версия Java требуется?** Поддерживается любой JDK 8 и выше.  
- **Можно ли конвертировать OneNote в PNG?** Конечно; API позволяет сохранять страницы как изображения.  
- **Безопасен ли процесс в многопоточной среде?** Установка локали по умолчанию является глобальной, поэтому её следует выполнить один раз при запуске приложения.

## Что такое «установить локаль по умолчанию» в Aspose.Note?
Установка локали по умолчанию сообщает Aspose.Note, какой язык и культурные соглашения применять при разборе дат, чисел и текста. Это важно для многорегиональных приложений, которым требуется единообразное форматирование для разных пользовательских локалей.

## Почему стоит устанавливать локаль по умолчанию при работе с OneNote?
- **Точная репрезентация данных:** Даты и числа отображаются корректно для целевой аудитории.  
- **Последовательные строки UI:** Текст, извлечённый из OneNote, учитывает языковые настройки.  
- **Упрощённая конвертация:** При последующей конвертации файла OneNote в PNG или другие форматы визуальное оформление будет соответствовать выбранной локали.

## Требования

- **Среда разработки Java:** Установлен JDK и настроена переменная `JAVA_HOME`.  
- **Библиотека Aspose.Note:** Скачайте последнюю JAR‑файл по [ссылке для загрузки](https://releases.aspose.com/note/java/).  
- **Действительный файл лицензии Aspose** (для тестов подходит бесплатная trial‑версия).

## Импорт пакетов

Перед написанием кода импортируйте классы, предоставляющие необходимый функционал.

```java
import java.io.IOException;
import java.nio.file.Path;
import java.util.Locale;
import com.aspose.note.Document;
import com.aspose.note.License;
import com.aspose.note.LocaleOptions;
```

## Шаг 1: Установить лицензию Aspose

```java
License license = new License();
license.setLicense("licenseFile");
```

Установка лицензии Aspose разблокирует все возможности, включая работу с локалью и конвертацию изображений.

## Шаг 2: Установить локаль по умолчанию

```java
java.util.Locale.setDefault(new java.util.Locale("en", "rs"));
```

Здесь мы **устанавливаем локаль по умолчанию** на английский (`en`) с кодом страны `rs`. При необходимости замените коды языка и страны на соответствующие вашей целевой локали.

## Шаг 3: Загрузить документ OneNote

```java
String inputFile = "Sample1.one";
Path inputPath = Paths.get(inputFile);

Document oneFile = new Document(inputPath.toString());
```

Этот шаг **загружает документ OneNote** в объект `Document`, чтобы вы могли работать с его содержимым.

## Шаг 4: Конвертировать OneNote в PNG (конверсия файла OneNote)

```java
oneFile.save("sample.png");
```

Метод `save` выполняет **конверсию файла onenote**, экспортируя блокнот (или отдельную страницу) в виде изображения PNG — по сути **convert onenote to png** и **save onenote as image**.

## Распространённые проблемы и советы

- **Лицензия не найдена:** Убедитесь, что путь к `licenseFile` указан правильно и файл имеет права чтения.  
- **Локаль не применяется:** Вызовите `Locale.setDefault` **до** загрузки документа.  
- **Отсутствует вывод изображения:** Проверьте, что файл OneNote действительно содержит рендеримые страницы; пустые блокноты приведут к пустому PNG.
