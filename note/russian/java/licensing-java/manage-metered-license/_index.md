# Конвертировать OneNote в PDF и управлять лицензией с оплатой по потреблению в Java

## Введение

В этом руководстве вы узнаете, как **конвертировать OneNote в PDF**, с помощью лицензии с оплатой по требованию с библиотекой Aspose.Note для Java. Лицензия с оплатой по потреблению позволяет отслеживать расходование кредитов в первое время, предоставляя гибкую оплату только за использованное. Мы проходим весь процесс — от настройки ключей доступа до загрузки файла OneNote, конвертации его в PDF и проверки использования кредитов.

## Быстрые ответы
- **Что означает «конвертировать OneNote в PDF»?** Это преобразует файл .one в переносимый PDF-документ.
- **Какая библиотека выполняет преобразование?** Aspose.Note для Java предоставляет простой API для этой задачи.
- **Нужна ли мне лицензия для конвертации?** Да, для использования в продакшене требуется лицензия с оплатой по потреблению.
- **Как я могу отслеживать использование?** Используйте `Metered.getConsumptionCredit()` и `Metered.getConsumptionQuantity()`.
- **Требуется ли дополнительная настройка?** Достаточно Java JDK и JAR‑файла Aspose.Note.

## Что такое «конвертировать OneNote в PDF»?

Конвертация OneNote в PDF создает статическое, широко применяемое представление страниц вашего блокнота. PDF‑файлы идеально подходят для обмена, печати или архивирования, а также для сохранения макетов, шрифтов и изображений на разных платформах.

## Зачем использовать лимитную лицензию для этого преобразования?

Лицензия с оплатой по потреблению взимает плату на основе фактического расхода кредита, а не фиксированной суммы. Эта модель идеальна, когда вы периодически совершаете конвертации или хотите недорого контролировать расходы, получая при этом полный функционал Aspose.Note.

## Предварительные условия

Перед тем как приступить, убедитесь, что у вас есть:

1. **Java Development Kit (JDK)** – любая современная версия (рекомендовано JDK11+). скачать ее [здесь](https://www.oracle.com/java/technologies/javase-jdk11-downloads.html).
2. **Aspose.Note для библиотеки Java** – предоставить полную JAR‑библиотеку с [официального сайта](https://releases.aspose.com/note/java/).

> **Совет для профессионалов:** Добавьте JAR-файл Aspose.Note в путь к классам вашего проекта или воспользуйтесь сборкой системы, такой как Maven/Gradle, для управления зависимостями.

## Импорт пакетов

Сначала импортируйте необходимые классы. Сохраните этот блок точно как показано — никаких изменений в коде.

```java
import com.aspose.note.Document;
import com.aspose.note.Metered;



import java.nio.file.Paths;
```

## Шаг 1: Настройка лимитированной лицензии

Здесь мы создаём объект `Metered` и передаём публичный/приватный ключи, полученные от Aspose. Этот шаг активирует режим лицензии с оплатой по потреблению для всех последующих операций, включая вызов **convert OneNote to PDF**.

```java
Metered metered = new Metered();
metered.setMeteredKey("MyPublicKey", "MyPrivateKey");
```

## Шаг 2: Загрузка документа OneNote и преобразование в PDF

```java
String dataDir = "Your Document Directory";
Document doc = new Document(Paths.get(dataDir,"Sample1.one").toString());
doc.save(Paths.get(dataDir,"MeteredLicense.pdf").toString());
```

- **Load**: `Document` читает файл `.one`, расположенный в `dataDir`. Это классический способ **load OneNote document** для обработки.  
- **Convert & Save**: Вызов `save` с расширением `.pdf` автоматически **convert .one to .pdf**. Операция также **saves PDF from OneNote** в той же папке. Замените `"Your Document Directory"` реальным путём на вашей машине.

## Шаг 3: Проверка расхода кредитов до и после преобразования

```java
System.out.println(String.format("Credit before operation: %.02f", Metered.getConsumptionCredit()));
System.out.println(String.format("Consumption quantity before operation: %.02f", Metered.getConsumptionQuantity()));
```

Эти две строки выводят оставшийся баланс кредитов и количество кредитов, использованных при конвертации. Выполнение тех же вызовов после операции `save` покажет обновлённые значения, позволяя убедиться, что лицензия с оплатой по потреблению корректно отслеживает использование.

## Распространенные проблемы и решения

| Выпуск | Почему это происходит | Исправить |
|-------|----------------|-----|
| **Неверные ключи лицензии с оплатой по потреблению** | Ключи были сделаны с ошибкой или потеряны. | Проверьте ключи в своей учётной записи Aspose; Необходимо при сгенерировании новых. |
| **Файл не найден** | Путь `dataDir` указан неверно. | Используйте абсолютный путь или проверьте относительный путь от начала проекта. |
| **Недостаточно кредитов** | Все кредиты израсходованы. | Получите дополнительные кредиты или продлите постоянную лицензию для решения задач с высоким объемом. |
